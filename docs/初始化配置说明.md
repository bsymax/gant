# 初始化配置：人员、项目、时间

## 1. 思路

- 在**不入 Git 的本地文件**里写好「团队与项目初值」→ 执行 **Prisma 种子**写入 SQLite。  
- 这样你可以先在表格或文本文案里**商量好**名单，再落库；不必在网页里逐个点。

## 2. 文件位置与命名

1. 复制：  
   `web/config/initial-data.example.json`  
   →  
   `web/config/initial-data.json`
2. **只编辑后者**。根目录 `.gitignore` 已忽略 `web/config/initial-data.json`，**不会**被 Git 提交（避免把真实人名/邮箱推上远端）。

## 3. 字段说明（与 JSON 一一对应）

### users[]

| 字段 | 必填 | 说明 |
|------|------|------|
| email | 是 | 系统内唯一键，建议固定域名如 `@local.dev` |
| name | 是 | 显示名 |
| role | 是 | `LEAD`（仅一名组长在业务上最清晰）或 `MEMBER` |

### projects[]

| 字段 | 必填 | 说明 |
|------|------|------|
| title | 是 | 项目名；种子**用标题去重**，已存在则**跳过**该项目，避免重复 |
| description | 否 | 说明 |
| scheduleMode | 是 | `IN_RESERVE` 储备 / `ON_TIMELINE` 已上轴 / `TBD_ON_TIMELINE` 在轴上但日期待定 |
| status | 否 | 默认 `ACTIVE`；另有 `DRAFT` / `DONE` / `ARCHIVED` |
| plannedStart / plannedEnd | 否 | **仅**在 `ON_TIMELINE` 时建议填写；`YYYY-MM-DD` |
| memberEmails | 是 | 参与人邮箱列表，须已在 `users` 中定义 |
| tasks | 否 | 事项列表 |
| tasks[].title | 是 | 若写 tasks |
| tasks[].status | 否 | 如 `TODO` / `IN_PROGRESS` / `DONE` |
| tasks[].assigneeEmails | 否 | 协作人邮箱 |

## 4. 如何应用

在 **`web` 目录**下：

```bash
npm run db:seed
```

- 会 **upsert** 所有 `users`（按 `email`）。  
- 对每个 `project`：若**同名标题已存在**则**跳过**该项目。  
- 未放置 `config/initial-data.json` 时，仍使用**内置默认演示**（7+1 与三个示例项目）。

**全新库**（可清空重来时）：

```bash
# 会删库，慎用；开发机常用
rm -f prisma/dev.db prisma/dev.db-journal
npx prisma migrate dev
# migrate 会跑 seed
```

## 5. 与「表格版」的对应

若你更习惯在 **Word/Excel/飞书文档** 里用表格写，可对照 [初始化配置表-模板](./初始化配置表-模板.md) 填一版，**再手抄**到 `initial-data.json`（或请助手代抄）。字段名在表里与 JSON 一致，避免歧义。

## 6. 常见问题

- **Q：改完 JSON 没生效？**  
  确认路径是 `web/config/initial-data.json`；且「同名项目会跳过」——若只改已存在项目的内容，应去 **网页里改**或先删该条项目/换标题再 seed。  
- **Q：能否只加用户不加项目？**  
  可以，`projects` 写 `[]` 即可。  
- **Q：与网页里后改的数据会冲突吗？**  
  seed 不会删你在网页里新增的数据，只对文件里**新项目**做「不存在则建」；用户行始终 upsert 覆盖**同名 email** 的 `name/role`。
