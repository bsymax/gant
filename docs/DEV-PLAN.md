## 1. 本阶段我判断「先落地」的功能（V1.0 代码已覆盖或占位）

| 模块 | 范围 | 说明 |
|------|------|------|
| **工程与版本** | Git + 语义化 `package.json` 版本 + `CHANGELOG` | 代码与需求可追溯 |
| **本地运行** | SQLite 单文件 + `prisma migrate` | 无外部依赖，符合「先本地」 |
| **用户与角色** | 1 组长 + 7 成员 种子数据；登录（开发期「选人即登录」） | 可替换为真实鉴权不推翻模型 |
| **§3.1 A+B** | 储备池 CRUD、**上战场**（入甘特+日期或入 TBD）、**降回探索** | 核心业务状态机 |
| **项目 / 事项** | 项目 + 事项两级、事项多对多**协作人**、**任一人可改** | 与 PRD 一致 |
| **版本记录（产品）** | `AuditLog`：项目/事项**创建、更新、状态流转、协作人变更**等 | 满足「变更历史可查证」最低线 |
| **UI 基调** | 暗色、网格/HUD 轻量、指挥台布局；甘特**首版为周粒度时间条占位** | 可交互迭代为真拖拽、真像素级甘特 |

**刻意延后**（不阻塞 V1 合入，下一迭代）：精确拖拽甘特、飞书/企微、能力树、业绩字段、多组 API 实装。

## 2. 需要 Git / 工程版本控制的内容

| 类别 | 是否纳入 Git | 备注 |
|------|----------------|------|
| 应用源码、`prisma/schema.prisma`、迁移脚本 | 是 | 主分支可审查 |
| `docs/*.md`（含 PRD、本文） | 是 | 与发版说明交叉引用 |
| `CHANGELOG.md`、版本号 | 是 | 与 `package.json` 版本一致策略 |
| `.env`、`.env.local`、SQLite 数据文件 | **否**（只提交 `.env.example`） | `dev.db` 在 `.gitignore` |
| `node_modules`、`.next` | 否 | 标准 ignore |

**产品内「版本管理」**（非 Git）：`AuditLog` 表，记录业务变更；以后如需「回滚到某历史版本」可基于该表做 UI。
