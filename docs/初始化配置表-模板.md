# 初始化配置表（模板）

> 将下列表格复制到本机文档中填写，确认无误后，可**转抄**为 `web/config/initial-data.json`，或直接维护 **`web/config/initial-csv/*.csv`** 后执行 `npm run config:from-csv`。**日期**建议统一 `YYYY-MM-DD`。**人员一律用 ERP 工号作唯一键**（与 `users.erp` 一致）。

---

## 表 A — 用户（与 JSON `users` / `users.csv` 对应）

| ERP (erp) | 显示名 (name) | 角色 (role) |
|-----------|---------------|-------------|
| 8800001 | 周岚 | LEAD |
| 8800101 | 白屿 | MEMBER |
|  |  |  |

*角色只能填：`LEAD` 或 `MEMBER`。*

---

## 表 B — 项目（与 `projects` 对应）

| 项目标题 (title) | 说明 (description) | 排期模式 (scheduleMode) | 计划开始 (plannedStart) | 计划结束 (plannedEnd) | 项目成员 (memberErps，逗号分隔) |
|------------------|--------------------|------------------------|--------------------------|------------------------|----------------------------------|
|  |  | IN_RESERVE / ON_TIMELINE / TBD_ON_TIMELINE | 可空 | 可空 | 如 8800001,8800101 |

* `IN_RESERVE`：储备池，可不填日期。  
* `ON_TIMELINE`：建议填 `plannedStart`；结束可空。  
* `TBD_ON_TIMELINE`：已上轴但日期待定，不填起止。  

*同一「标题」在数据库里已存在时，**种子会跳过**该项目，避免重复。*

---

## 表 C — 某项目下的事项（嵌套在表 B 对应项目的 `tasks`）

（若不用种子建事项，可只填表 A、B。）

| 所属项目标题 | 事项标题 (title) | 状态 (status) | 协作人 (assigneeErps，逗号分隔) |
|--------------|-------------------|--------------|----------------------------------|
|  |  | TODO / IN_PROGRESS / DONE 等 | 如 8800102,8800103 |

---

**抄成 JSON 时**：`memberErps` / `assigneeErps` 为 **字符串数组** `["8800001","8800102"]`。**用 CSV 时**见 `config/initial-csv` 中各列名，勿与 JSON 列混用（脚本会生成 JSON）。
