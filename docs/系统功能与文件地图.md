# 系统功能与文件地图

> 版本 **0.2.0** 起适用。你若是**想改某一块功能**，在表里找到「建议编辑的文件」即可（路径均相对仓库根目录，以 `web/` 为主）。

## 1. 功能一览

| 功能 | 用户能做什么 | 设计要点（对应 PRD） |
|------|----------------|----------------------|
| 登录 | 开发期在 `/login` **选用户**即登录 | `AUTH_DEV=1` 时无密码，生产应换真鉴权 |
| 指挥台 | 看**储备池**、**8 周时间轴**、**TBD 待定区**；新建项目进池；对池内项目**上战场**；从轴**降回探索** | PRD §3.1 已定稿 A+B |
| 项目详情 | 改标题/说明、**排兵(成员)**、**事项**、看**项目级审计** | 协作人/组长可改；审计可追溯 |
| 数据落库 | 用户、项目、成员关系、事项、任务协作、审计行 | Prisma + SQLite 本地 `web/prisma/dev.db` |
| 配置导入 | 表格 `initial-csv/*.csv` 或 JSON 一次写入库 | [初始化配置说明](./初始化配置说明.md)、[CSV 流程](./CSV初始配置-方案与流程.md) |

## 2. 路由与页面

| 路径 | 文件 | 说明 |
|------|------|------|
| `/` | `web/app/page.tsx` | 有会话则进指挥台，否则进登录 |
| `/login` | `web/app/(auth)/login/page.tsx` | 选用户、提交登录 |
| `/command` | `web/app/command/page.tsx` | 主指挥台 + 调 `CommandGantt` |
| `/command/projects/[id]` | `web/app/command/projects/[id]/page.tsx` | 单项目维护 |
| 指挥台顶栏、鉴权 | `web/app/command/layout.tsx` | 未登录会重定向 `/login` |

## 3. 服务端行为（表单子动作）

| 行为 | 文件 | 主要函数/说明 |
|------|------|----------------|
| 登录/退出 | `web/app/actions/auth.ts` | `loginAction` / `logoutAction` + Cookie 会话 |
| 项目：建池、上战场、降回、改标题、加人 | `web/app/actions/projects.ts` | 与 `ScheduleMode` 状态机一致 |
| 事项 | `web/app/actions/tasks.ts` | 创建/状态/协作人 |
| 写审计行 | `web/lib/audit.ts` | `logAudit` |
| 谁能改什么 | `web/lib/permissions.ts` | `canAccessProject` / `canCreateTask` 等 |

## 4. 展示组件

| 组件 | 文件 | 说明 |
|------|------|------|
| 甘特 + TBD 区 | `web/components/CommandGantt.tsx` | 8 周条带 + 条的位置计算 |

## 5. 身份与数据

| 主题 | 文件 | 说明 |
|------|------|------|
| 会话 | `web/lib/auth.ts` | JWS Cookie；用户展示 **ERP**（`User.erp`） |
| 常量（状态/角色字符串） | `web/lib/constants.ts` | 与 Prisma 里 `String` 列一致 |
| 数据库访问 | `web/lib/prisma.ts` | 单例 `prisma` |
| 表结构、迁移 | `web/prisma/schema.prisma`、`web/prisma/migrations/` | 用户**唯一外显键**为 `User.erp`（非 email） |
| 种子 / 初值 | `web/prisma/seed.ts`、`web/config/initial-data.json`、`web/config/initial-csv/*.csv` | 见 [初始化配置说明](./初始化配置说明.md) |
| CSV → JSON | `web/scripts/csv-to-initial-data.ts` | `npm run config:from-csv` |
| 周窗计算 | `web/lib/dates.ts` | 甘特用「本周起 8 周」 |

## 6. 界面风格（新配色 v2）

| 要改什么 | 去哪里 |
|----------|--------|
| **总配色、亮暗、对比度、按钮/面板/输入框** | **`web/app/globals.css`**（优先改 `:root` 与 `.gant-*`） |
| 个别页面上还残留的 Tailwind 色类 | 对应 `page.tsx` / `CommandGantt.tsx` |

细读：[界面与配色-极光指挥台v2](./界面与配色-极光指挥台v2.md)。

## 7. 版本与记录

| 概念 | 说明 |
|------|------|
| **Git** | 代码与迁移脚本；日常你可只**备份整个文件夹**，不必会 Git。 |
| **CHANGELOG** | 根目录 `CHANGELOG.md` 记产品/界面版本。 |
| **应用内「版本」** | 数据库 `AuditLog`：谁在何时改了什么；不是 Git 回退。 |
